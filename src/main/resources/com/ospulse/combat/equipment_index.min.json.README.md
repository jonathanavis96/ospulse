# equipment_index.min.json — provenance & regeneration

A compact `{ "<itemId>": ["<name>", <slotOrdinal>, <isTwoHanded>] }` map: every
equippable item id that also has a row in `equipment_stats.min.json` (i.e. the
DPS engine can actually price/compute it), paired with its display name,
`EquipmentInventorySlot` ordinal (0=head 1=cape 2=neck/amulet 3=weapon 4=body
5=shield 7=legs 9=hands/gloves 10=feet/boots 12=ring 13=ammo) and a two-handed
flag (weapon-slot entries only; always `false` elsewhere).

This is the missing piece for the Phase 2/3 item search + optimiser candidate
pool: `equipment_stats.min.json` (OSRS-cache-derived) has bonuses but no names,
and deliberately omits `isTwoHanded` (its README: "no cache param reliably
encodes it"). This file supplies name + slot + 2h for the same id space,
filtered to the overlap so every entry here is guaranteed computable — and,
crucially, lets the EDT-only what-if picker (`GearSection`) resolve 2h/shield
exclusivity for a hypothetical weapon WITHOUT an `ItemManager` call (which
would assert the client thread and crash off it).

## Source
weirdgloop `osrs-dps-calc` `cdn/json/equipment.json` (same upstream already
used for `weapon_categories.min.json` — see that file's README for the
license note: GPLv3 code, OSRS Wiki CC-BY-NC-SA underlying data; same
data-hygiene TODO applies here).

## Regeneration
```bash
curl -sL https://raw.githubusercontent.com/weirdgloop/osrs-dps-calc/main/cdn/json/equipment.json -o /tmp/equipment.json
python3 - <<'PY'
import json
with open('/tmp/equipment.json') as f:
    equip = json.load(f)
with open('equipment_stats.min.json') as f:
    stats_ids = set(int(k) for k in json.load(f).keys())
SLOT_MAP = {'head':0,'cape':1,'neck':2,'weapon':3,'body':4,'shield':5,
            'legs':7,'hands':9,'feet':10,'ring':12,'ammo':13}
index = {}
for item in equip:
    iid, name, slot = item.get('id'), item.get('name'), item.get('slot')
    if iid is None or name is None or slot is None:
        continue
    ordinal = SLOT_MAP.get(slot)
    if ordinal is None or iid not in stats_ids or str(iid) in index:
        continue
    index[str(iid)] = [name, ordinal, bool(item.get('isTwoHanded', False))]
with open('equipment_index.min.json', 'w') as f:
    json.dump(index, f, separators=(',', ':'))
PY
```
Re-run whenever `equipment_stats.min.json` is regenerated so the two stay in
sync (an id present in one but not the other is silently dropped from search).

Items in file: 3179 (of 5327 upstream equipment.json entries; the rest have no
matching cache-derived stats row and are excluded). Two-handed weapons: 328.
