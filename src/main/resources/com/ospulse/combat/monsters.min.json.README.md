# monsters.min.json — data provenance

Source: https://raw.githubusercontent.com/weirdgloop/osrs-dps-calc/main/cdn/json/monsters.json
(`weirdgloop/osrs-dps-calc`, **GPL-3.0** licensed *data* file — game facts, not
calculation code. This plugin's `formula`/effects logic is written
independently from public OSRS Wiki maths and does not vendor any GPL code;
only these plain game-data facts (monster hp/defence/attributes) are reused,
which is the same public information published on the OSRS Wiki itself.)

Trimmed to the fields `com.ospulse.combat.Monster` actually needs (id, display
name, hitpoints, per-style defence bonuses, magic level, size, a mapped
subset of attributes, attack speed) — 2.3&nbsp;MB -> ~550&nbsp;KB, no startup
network cost.

## Field mapping (source -> trimmed)

| trimmed field     | source path                                  |
|--------------------|-----------------------------------------------|
| `id`               | `id`                                          |
| `name`             | `name` + `" (" + version + ")"` if `version` non-empty |
| `hitpoints`        | `skills.hp`                                   |
| `defenceLevel`     | `skills.def`                                  |
| `dstab`            | `defensive.stab`                              |
| `dslash`           | `defensive.slash`                             |
| `dcrush`           | `defensive.crush`                             |
| `dmagic`           | `defensive.magic`                             |
| `drange`           | `defensive.standard` (ranged defence vs standard-weight ammo; the source splits this into heavy/standard/light by ammo type — Tier-A simplification uses the standard value uniformly; refining to ammo-type-specific ranged defence is a Tier-B extension) |
| `magicLevel`       | `skills.magic`                                |
| `size`             | `size`                                        |
| `attributes`       | `attributes`, mapped: `undead->UNDEAD`, `dragon->DRAGON`, `demon->DEMON`, `kalphite->KALPHITE`, `vampyre1/2/3->VAMPYRE`, `leafy->LEAFY`, `shade->SHADE`. Unmapped source attributes (`fiery`, `flying`, `golem`, `penance`, `rat`, `spectral`, `xerician`) are dropped — they gate no Tier-A effect today. |
| `attackSpeedTicks` | `speed`                                       |
| `weakness`         | `weakness` (object `{element, severity}`), mapped: `air->WIND`, `water->WATER`, `earth->EARTH`, `fire->FIRE` (`Spell.Element` names — upstream calls the wind element "air"). Omitted entirely when the source has no weakness (`element` absent/`"none"`, or `severity` <= 0), so `Monster.weaknessElement()` stays `null` for those monsters. |

`demonbaneResistPercent` is a **hand-patched field, NOT part of this
minifier** (currently only Duke Sucellus's 3 encounter variants, npc id
12191/12195, at 30) — regenerating from upstream must re-apply these
overrides by npc id or they are silently lost. See `Monster#demonbaneResistPercent()`.

Each source entry (one per id+version combination) becomes one `Monster`
record with a single-element `npcIds` list — the source does not pre-group
multiple RuneLite NPC ids under one combat-stat record.

## Regenerating

```bash
curl -sL -o /tmp/monsters_full.json \
  https://raw.githubusercontent.com/weirdgloop/osrs-dps-calc/main/cdn/json/monsters.json
python3 - <<'PY'
import json

ATTR_MAP = {
    'undead': 'UNDEAD', 'dragon': 'DRAGON', 'demon': 'DEMON', 'kalphite': 'KALPHITE',
    'vampyre1': 'VAMPYRE', 'vampyre2': 'VAMPYRE', 'vampyre3': 'VAMPYRE',
    'leafy': 'LEAFY', 'shade': 'SHADE',
}
ELEMENT_MAP = {
    'air': 'WIND', 'water': 'WATER', 'earth': 'EARTH', 'fire': 'FIRE',
}
# Hand-patched demonbane-resistance overrides (NOT in the upstream data) —
# preserved across regeneration by npc id. Extend this map if more are added.
DEMONBANE_RESIST_OVERRIDES = {
    12191: 30,  # Duke Sucellus (Awakened/Post-quest, Awake)
    12195: 30,  # Duke Sucellus (Quest, Awake)
}
with open('/tmp/monsters_full.json') as f:
    data = json.load(f)
out = []
for m in data:
    skills = m.get('skills') or {}
    defensive = m.get('defensive') or {}
    version = m.get('version') or ''
    name = f"{m['name']} ({version})" if version else m['name']
    attrs = sorted({ATTR_MAP[a] for a in (m.get('attributes') or []) if a in ATTR_MAP})
    entry = {
        'id': m['id'], 'name': name,
        'hitpoints': skills.get('hp', 1), 'defenceLevel': skills.get('def', 0),
        'dstab': defensive.get('stab', 0), 'dslash': defensive.get('slash', 0),
        'dcrush': defensive.get('crush', 0), 'dmagic': defensive.get('magic', 0),
        'drange': defensive.get('standard', 0),
        'magicLevel': skills.get('magic', 1), 'size': m.get('size', 1),
        'attributes': attrs, 'attackSpeedTicks': m.get('speed'),
    }
    weakness = m.get('weakness') or {}
    mapped_element = ELEMENT_MAP.get(weakness.get('element'))
    severity = weakness.get('severity') or 0
    if mapped_element and severity > 0:
        entry['weakness'] = {'element': mapped_element, 'severity': severity}
    resist = DEMONBANE_RESIST_OVERRIDES.get(m['id'])
    if resist:
        entry['demonbaneResistPercent'] = resist
    out.append(entry)
with open('src/main/resources/com/ospulse/combat/monsters.min.json', 'w') as f:
    json.dump(out, f, separators=(',', ':'))
PY
```

## Shipping as gzip

The plugin bundles this data gzip-compressed
(`monsters.min.json.gz`, ~560KB -> ~52KB) to keep the Hub jar small;
`MonsterRepository` decompresses it transparently at load time with
`GZIPInputStream`. Only the `.gz` is bundled as a resource — the plain
`.json` above is a regeneration intermediate, not shipped.

After regenerating the plain JSON with the script above, produce the `.gz`
deterministically (no embedded mtime, so the output is byte-identical on
every machine/run given the same input):

```bash
gzip -n -9 -c src/main/resources/com/ospulse/combat/monsters.min.json \
  > src/main/resources/com/ospulse/combat/monsters.min.json.gz
rm src/main/resources/com/ospulse/combat/monsters.min.json
```
