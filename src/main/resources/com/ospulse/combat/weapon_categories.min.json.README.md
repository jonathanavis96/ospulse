# weapon_categories.min.json — provenance & regeneration

A trimmed `{ "<itemId>": "<weapon-category>" }` map: each equippable **weapon-slot**
item id → its OSRS weapon category (the in-game combat-options "weapon type",
lower-cased). The category is the *only* thing needed to know a weapon's attack
styles — RuneLite's `ItemEquipmentStats` exposes attack bonuses and speed but
**not** the weapon category, so it must be bundled.

Consumed by `com.ospulse.combat.WeaponCategoryRepository` → `WeaponStyle` /
`WeaponStyles.forCategory(...)`, which drive the Gear-DPS panel's ranked
attack-style picker (and, later, the Phase 2/3 optimiser's "styles of a weapon
you don't own yet" lookup — same id→category path works for any weapon id, not
just the equipped one).

## Source
Derived from **weirdgloop `osrs-dps-calc`** `cdn/json/equipment.json`
(<https://github.com/weirdgloop/osrs-dps-calc>), the same upstream our monster
snapshot came from. The `{name, type, stance}` style tables themselves are ported
from that repo's `getCombatStylesForCategory` (`src/utils.ts`). Category strings
match `src/enums/EquipmentCategory.ts`.

## Regeneration
```bash
curl -sL https://raw.githubusercontent.com/weirdgloop/osrs-dps-calc/main/cdn/json/equipment.json \
  | jq -c '[.[] | select(.slot=="weapon") | {(.id|tostring): (.category|ascii_downcase)}] | add' \
  > weapon_categories.min.json
```
Re-run on a data-refresh PR whenever new weapon types ship (see the update-treadmill
note in the Technical doc). Currently **1566** weapon ids across **31** categories.
